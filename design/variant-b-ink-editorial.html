How I ship a working PWA in 10 days

June 10, 2026 · 6 min read · Neil Busque

Most side projects die in week two. Mine used to as well, until I made the process so boring that finishing became the default outcome.

Day one is always the same: schema first, then the read path. Nothing renders until the data shape is locked.

// the only client I allow myself on day one
const { data } = await supabase
  .from('posts')
  .select('*')
  .eq('status', 'published')
  .order('published_at', { ascending: false });

By day three the app does one real thing end to end. Everything after that is compounding, not gambling.
